# CipherNest — Quick Start Guide

## 🚀 Начало работы за 3 шага

### 1. Получите токен бота

1. Откройте Telegram и найдите `@BotFather`
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте токен бота (выглядит так: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

### 2. Настройте конфигурацию

```bash
# Скопируйте шаблон конфигурации
cp .env.example .env

# Откройте .env и вставьте ваш токен
# Windows:
notepad .env

# Linux/Mac:
nano .env
```

Добавьте ваш токен:
```env
BOT_TOKEN=ваш_токен_здесь
```

### 3. Запустите бота

```bash
python main.py
```

Вы должны увидеть:
```
2026-04-03 12:00:00 - root - INFO - CipherNest Bot starting...
2026-04-03 12:00:01 - aiogram.dispatcher - INFO - Start polling
```

Бот работает! Откройте его в Telegram и отправьте `/start`.

## 📱 Как использовать бота

### Базовые операции

1. **Отправьте `/start`** — появится главное меню
2. **Выберите категорию** (например, «🔤 Кодирование»)
3. **Выберите алгоритм** (например, «Base64»)
4. **Выберите операцию** (например, «▶️ Кодировать»)
5. **Отправьте текст** и получите результат мгновенно!

### Примеры использования

#### Кодирование текста в Base64
```
/start → 🔤 Кодирование → Base64 → ▶️ Кодировать
Отправьте: Hello World
Результат: SGVsbG8gV29ybGQ=
```

#### Шифрование секретного сообщения
```
/start → 🔐 Симметричные → AES-256-GCM → 🔐 Зашифровать
Отправьте: My secret message
Результат: {"salt": "...", "nonce": "...", "ciphertext": "..."}
```

#### Хеширование пароля
```
/start → 🧮 Хеши → Argon2id → 🔒 Хеш
Отправьте: my_secure_password
Результат: {"salt": "...", "hash": "..."}
```

#### Генерация RSA ключей
```
/start → 🔑 Асимметричные → RSA-2048 → 🔑 Сгенерировать ключи
Результат: Приватный + публичный ключ (отправляются файлами)
```

### Продвинутые функции

#### 🔗 Режим цепочки
Примените несколько преобразований последовательно:

```
/chain
Отправьте: base64
Отправьте: rot13
Отправьте: hex
Отправьте: готово
Затем отправьте ваш текст
Результат: Текст пройдёт Base64 → ROT13 → Hex
```

#### 🔍 Автодетект
Просто отправьте закодированный текст — бот распознает его:

```
Отправьте: SGVsbG8gV29ybGQ=
Бот: 🔮 Похоже на BASE64 (уверенность: 90%)
     Хотите декодировать base64?
```

#### 🎓 Режим обучения
Узнайте об алгоритмах:

```
/start → 🎓 Обучение
Отправьте: aes_gcm
Результат: Полное описание, история, применение, информация о безопасности
```

## 🛡️ Правила безопасности

✅ **Рекомендуется:**
- Использовать для изучения и экспериментов
- Тестировать алгоритмы перед внедрением в код
- Генерировать примеры ключей для разработки

❌ **Не рекомендуется:**
- Отправлять настоящие пароли или production-ключи
- Использовать классические шифры (ROT13, Цезарь) для реальной защиты
- Применять MD5/SHA-1 в security-критичных приложениях

## ⚙️ Настройки

### Переменные окружения

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `BOT_TOKEN` | Токен Telegram-бота (обязательно) | — |
| `LOG_LEVEL` | Уровень логирования | `INFO` |

### Rate Limiting

По умолчанию: **10 запросов за 60 секунд** на одного пользователя

Для изменения откройте `app/bot_handlers.py`:
```python
rate_limiter = RateLimiter(max_requests=10, window_seconds=60)
```

## 🐛 Решение проблем

### Бот не отвечает
- Проверьте, правильно ли указан `BOT_TOKEN` в `.env`
- Убедитесь, что есть подключение к интернету
- Проверьте консоль на наличие ошибок

### Ошибка «Module not found»
```bash
pip install -r requirements.txt
```

### Ошибки импорта
Убедитесь, что запускаете из директории проекта:
```bash
cd путь/к/вашей/папке/ciphernest
python main.py
```

## 💡 Советы

- **Длинные результаты** (>4096 символов) отправляются файлами `.txt`
- **Кнопка копирования** доступна для удобного копирования результатов
- **Файлы до 20 МБ** поддерживаются
- **Юникод/эмодзи** полностью поддерживаются
- **Режим цепочки** поддерживает неограниченное количество шагов
